# 🏢 Stage 6 — RFI (Request for Information)

## 🎯 Goal

Before asking for proposals, Acme first gathers:

Information about vendor capabilities

---

## ❓ What Is RFI (Request for Information)?

A Request for Information (RFI) is:

A document sent to vendors to understand their capabilities, experience, and approach, without asking for detailed pricing or full solution.

---

## 👥 Who Is Involved

### Customer (Acme)

| Role | Description |
|------|------------|
| Procurement | Leads the process |
| Enterprise Architects | Evaluate technical capability |
| IT Leadership | Provide strategic direction |

### Vendor

| Role | Description |
|------|------------|
| Sales / Pre-sales Teams | Manage response and positioning |
| Solution Architects | Provide technical inputs and approach |

---

## 🛠 What Happens

Acme sends RFI to multiple vendors (5–10 typically)

Vendors respond with:

| Response Area |
|---------------|
| Company profile |
| Experience: Cloud migrations |
| Experience: SAP / ERP transformations |
| High-level approach |
| Case studies |
| Certifications (e.g., Azure, AWS) |

---

## 📦 Output

| Output |
|--------|
| Shortlist of vendors (e.g., 2–4 vendors) |

👉 Only shortlisted vendors move to RFP stage
